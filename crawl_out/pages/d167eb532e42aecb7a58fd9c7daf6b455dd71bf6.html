Openlab Linux Cluster @ ICS

The ICS Instructional Openlab Linux Server cluster is designed for general-purpose computing at the School of ICS and is accessible to any user with an ICS Home Directory.

Openlab consists of 40-80 bare metal compute nodes running Ubuntu 22.04 LTS.

Students can connect to the cluster via any SSH client and will have access to the same ICS home directory as any other ICS managed Linux system and OpenHub @ ICS. This cluster is available off-campus via SSH with SSH keys when not connected to the campus VPN.

ğŸ”Œ Connecting to Openlab

You may log in remotely from any host using an SSH client.

For detailed setup instructions: Using VSCode with Openlab

Log into one of the following domain names:

Domain Name Operating System Purpose
openlab.ics.uci.edu Ubuntu 22.04 hosts General compute nodes
opengpu.ics.uci.edu Ubuntu 22.04 hosts GPU-enabled nodes

âš ï¸ Important: If you are running a long-running process, note the hostname you're using. The name openlab.ics.uci.edu is a load balancer and may not direct you to the same host twice.

Determine your hostname with: hostname

When returning to check your process, SSH directly to that specific hostname instead of openlab.ics.uci.edu.

ğŸŒ Off-Campus Access

Off-campus access requires SSH keys. Users connecting from off-campus cannot use passwords to log in to the Openlab cluster.

What ICS considers on-campus:

  • Any host with reverse DNS name ending in uci.edu (includes most campus hosts and VPN connections)
  • Any host connected to the OIT VPN

ğŸš‡ SSH Tunneling

To create an SSH tunnel to a specific host, you cannot use openlab.ics.uci.edu.

Example: User connected to Openlab, running Jupyter Lab on port 5555, on host circinus-2.ics.uci.edu: 

ssh -L 8080:localhost:5555 username@circinus-2.ics.uci.edu

âš–ï¸ Load Balancing

The name openlab.ics.uci.edu connects you to a load balancer that sends your connection to the machine with the lowest current load.

If your work requires multiple shells on the same host, log in directly using a hostname instead of openlab.ics.uci.edu.
