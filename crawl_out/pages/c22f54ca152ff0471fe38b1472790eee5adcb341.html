Tenable Nessus Agent

Tenable Nessus Agents are lightweight software programs that organizations can install on their systems to perform vulnerability and compliance assessments. This page contains instructions on how to install the package from the ICS custom repository, as well as links to OIT's directions.

External Documentation

  • Tenable Nessus Agent - Installation Guide
  • Tenable Nessus Vulnerability Scanner

Installing from ICS Custom Repo

Install the ICS Custom Repo

You may skip to Install the Package if you have already installed the ICS Custom Repo on your host.

Ubuntu 20.04 LTS and newer

Install the ICS custom repo on your Ubuntu host by running the following as root: 

cat << EOF >> /etc/apt/sources.list.d/ics-custom.list
deb [trusted=yes arch=amd64] https://repo.ics.uci.edu/ubuntu/ jammy main
EOF
apt update

CentOS7

Running the following block of code will add the ICS-custom repository, and set it disabled, to a CentOS7 host as root. 

cat << EOF >> /etc/yum.repos.d/ICS-custom.repo
[ICS-custom]  
name=CentOS-7 - ICS Custom Applications
baseurl=ftp://ftp.ics.uci.edu/pub/centos0/ics-custom-repo  
enabled=0
gpgcheck=0  
priority=10
EOF

Other Operating Systems

Please follow the directions in the Tenable Nessus Agent - Installation Guide documentation.

Unsupported Operating Systems

Important: The campus cybersecurity team will be contacting owners of end-of-life (EOL) operating systems that cannot run the EDR software. These users will need to either upgrade their systems or turn them off.

Install the Package

Once the ICS-custom repo has been installed, the following OS specific commands may be issued as root to install the package.

Ubuntu 20.04 LTS and newer

apt update 
apt install nessusagent

CentOS7

yum clean all 
yum --enablerepo=ICS-custom install NessusAgent

Configure

Important: It's important to include the ‚Äìgroups=ACA_ICS tag so that Helpdesk has access to the agent scans for your host 

/opt/nessus_agent/sbin/nessuscli agent link \
  --host=nessus.oit.uci.edu --port=443 \
  --key=84bd6d8fb397eab26eeac5da94090cd74601095515492d14e7658defaa702084 \
  --groups=ACA_ICS

Relinking

In order for the helpdesk team to read the scan reports from the Nessus client scans, the host must be part of the ACA_ICS groups. The following commands will relink your client to the Nessus scanner and ensure that it is assigned to the ACA_ICS groups.

Start by unlinking: 

/opt/nessus_agent/sbin/nessuscli agent unlink

Then link the agent again with the groups option: 

/opt/nessus_agent/sbin/nessuscli agent link \
  --host=nessus.oit.uci.edu --port=443 \
  --key=84bd6d8fb397eab26eeac5da94090cd74601095515492d14e7658defaa702084 \
  --groups=ACA_ICS

Start the Service

Issue the following command as root to ensure the service is enabled and started: 

systemctl enable nessusagent
systemctl start nessusagent

Confirm Installation

Systemctl

CentOS7 and Ubuntu 20.04 LTS and newer

Run the following systemctl command, as root, to confirm that the service is started. 

sudo systemctl status nessusagent

The output will appear similar to this:

Nessuscli

Run the nessucli command to check agent status, as root. 

sudo /opt/nessus_agent/sbin/nessuscli agent status
