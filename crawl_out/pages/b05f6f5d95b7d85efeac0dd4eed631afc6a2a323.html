Operating System Support Levels

Mission

Out mission is to provide researchers with a stable, supported, and robust Linux platform that allows them to perform their research unhindered. To that end, we have designed a few templates of various levels of Linux system autonomy and storage integration.

Types of Support

  • Managed with non-impersonating sudo
  • Managed with full sudo
  • Limited Support
  • Appliance Only
  • Roadmap

Note: Local data is never backed up. Please make sure you copy any important data into your home or extra directory.

LDAP Managed

Managed means that ICS Computing Support will:

  • Provision the operating system
  • Provide Single Sign-on via LDAP
  • Provide constant configuration and security updates via puppet
  • Service and resource Monitoring
  • ICS NFS home directories, extra directories and other storage servers.
  • Daily and DR backups of NFS storage.
  • Access to the ICS Software Library (/pkg).
  • Package installs from trusted OS repositories.
  • Compile new software to install in Software Library
  • Sudo for limited number of designated users excluding the commands that allow users to impersonate other users.
  • Service configuration and management (e.g. MySQL, Apache, MongoDB, Cycle Server)

Kerberos Managed

There is no material difference between Kerberos Managed and Keberos/Limited. See below.

LDAP/Limited

ICS Computing Support provides a platform that allows researchers to install and self-manage and maintain security of their own services. ICS Computing Support will perform best-effort support of OS and platform related issues. This model is a good fit for researchers that want to self-manage services that are built on packages that are already available as part of the distro such as Mysql or Apache2.

LDAP/Limited support has all of the features offered in LDAP Managed with the following caveats:

  • Best effort OS troubleshooting.
  • Additional package repositories can be installed by end user.
  • Service installation and configurations to be maintained by the end user.
  • Service security to be maintained by the end user.

Kerberos/Limited

ICS Computing Support to provide the platform for researchers that allows them to install, configure, and manage their own services. The is intended for researchers that require sudo privilege to run services that may be insecure in an NFS environment (such as docker or kubernetes).

Kerber/Limited comes with all the same support as LDAP/Limited with the following changes:

  • End users may have full sudo
  • ICS Storage Servers, home and extra directories available via NFS4 and Kerberos.
  • Docker, Vagrant, and VirtualBox are available.

Self Managed

Teams may choose to self manage systems. With certain caveats, so long as the equipment meats data center hosting requirements and can reasonably be supported in the ICS Data Center, ICS Computing Support will provide a network, power, and space in a cabinet.

Commonly teams choose to self manage because they want to make important decisions about installation and configuration or the equipment is uncommon and the ICS Computing Support doesn't have impactful experience or knowledge about the service the equipment will provide.

In all cases of self management, the team that owns the system will provide full support. This includes

  • OS Installation
  • OS Upgrade
  • Package Installation
  • Hardware troubleshooting
  • OS troubleshooting
  • Warranty report

Glossary

Full Sudo

Full Sudo privilege is delegated to the PI and one or two designated points of contact. These users can delegate that to additional group members.

Limited Sudo

Limited Sudo commands are enabled for PI and one or two designated points of contact. Commands are provided based on need

Most commands are available except commands that allow owner and user change (e.g. commands such `chown` and `su`). This restriction is due to the presence of NFS mounted directories.

Single Sign on

The system is attached to ICS LDAP servers providing a common sign-on, password, uid.

Network Storage

NFS

ICS Home and extra directories are available directly on systems with NFS access. NFS attached storage is not available on systems with full sudo due to irreconcilable security concerns.

FUSE

Network Storage via FUSE is available on any system.

Backups

Snapshots

Snapshot are kept up to 30 days on ICS /home and /extra directories. Only available and managed and limited types of support.

Backups

Backups are made to tape once per month for most /home and /extra/ directories. Available on managed and limited managed support.

OS Configuration

General configuration on all ICS managed systems is provided by the puppet several times a day. Puppet will automatically update things like /etc/hosts, grid engine, and the autofs configuration.

Self Install(Coming Soon)

Install media, kickstart, rescue disks and miscellaneous tools are all available via ICS PXE.

Software Library

ICS Computing Support routinely installs the latest version of popular packages for Managed platforms in the ICS Software Library. Additional packages available by request to helpdesk. Most software will work cross platform but it is not guaranteed (i.e. CentOS7 packages will frequently work under Ubuntu 18.04).

Virtual Environments

Virtual Box and Vagrant

See notes about VirtualBox/Vagrant. Vagrant is available only kerberos managed and kerberos/limited.

Docker

Docker is available only kerberos managed and kerberos/limited, and self managed due to lack of definable security.
