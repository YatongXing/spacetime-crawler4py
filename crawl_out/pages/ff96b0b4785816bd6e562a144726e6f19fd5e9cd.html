Trellix FireEye Endpoint Protection by OIT

Notes and Observations by pnimsomb

The anti-virus portion of FireEye seems to only be activated/enabled/installed for clients that have an ICS IP address range. By default (for the rest of the campus), FireEye is only set for threat detection, not with AV. Ryan Bakonis (before leaving UCI) mentioned there needs to be a common identifier for OIT Security to create a policy, so we settled on IP address range when Sophos licensing expired.

As of July 2022, the simplest way to get FireEye with the AV part installed/activated is to physically plug in the desktop/laptop being set up to an ICS managed ethernet connection. If the device has no ethernet port, try one of the USB-to-ethernet adapters. Alternatively, the WiFi within the Support suite area, ICS Zone, will also give an IP address within the ICS range. Setting up a computer using the main campus WiFi (UCInet Mobile Access or eduroam) will only enable the default threat detection portion. If a computer has already been setup with only the threat detection portion, then email security@uci.edu to request the AV portion be manually enabled/installed. OIT Security will need some form of identifier, so something like an IP address or hostname.

Self Installation

Linux

Installation

Note: Installation on managed hostsis handled by our Puppet or ansible configuration systems.

Installing from the ICS Custom Repo

Ubuntu 20.04+

Install the ICS custom repo on your Ubuntu host by running the following as root: 

cat << EOF >> /etc/apt/sources.list.d/ics-custom.list
deb [trusted=yes arch=amd64] https://repo.ics.uci.edu/ubuntu/ jammy main
EOF
apt update
CentSO7

Running the following block of code will add the ICS-custom repository, and set it disabled, to a CentOS7 host as root. 

cat << EOF >> /etc/yum.repos.d/ICS-custom.repo``[ICS-custom]  
name=CentOS-7 - ICS Custom Applications
baseurl=ftp://ftp.ics.uci.edu/pub/centos0/ics-custom-repo  
enabled=0
gpgcheck=0  
priority=10
EOF
Other Operating Systems

Please follow the directions in the Trellix Endpoint Security Installation documentation.

Install the Package

Once the ICS-custom repo has been installed, the following OS specific commands may be issued to install the package.

Ubuntu 20.04+
apt update 
apt install xagt
CentOS7
yum clean all 
yum --enablerepo=ICS-custom install xagt

Configure

In order to configure you client, please download the attached agent_config.json and then run the following command as root: 

/opt/fireeye/bin/xagt -i agent_config.json

Start the Service

Ensure the service has been started as root: 

 systemctl enable xagt systemctl start xagt

Confirm

CentOS7 and Ubuntu 20.04 LTS and newer

Execute systemctl commands as root: 

systemctl status xagt

The output will appear similar to this:

Notes (from Ryan Bakonis, rbakonis@uci.edu)

  The HX client software needs to talk to the managing cloud based server.  
  All updates, including the software, policies and protection rules as well as engines are updated via this mechanism.  
  This also allows us to track licensing as well. 
  Verify that your endpoint (server or workstation) has connectivity outbound to the internet to the required URL via ports 80/443. 
  URL: hexshj633-hx-agent-1.hex01.helix.apps.fireeye.com
  Conduct a software inventory and list any application to be whitelisted to OIT Security? 
  Common software to whitelist would be any existing antivirus software. 
  This will maximize performance and ensure compatibility with other antivirus software.

Installing Using BigFix

OIT manages the FireEye installer, so installation can be issued as a Fixlet. More details on how to issue a BigFix Fixlet is available in the Issuing a Fixlet via BigFix section.

Manual Installation Instructions

The install requires the following: 

  HX install package 
      Windows Version 33.46.00 (Jul, 2021) 
      
      **Main location:**  \\masterhit.ics.uci.edu\FireEye
      
      Alternative location (download from Webfiles):  [[https://webfiles.uci.edu/xythoswfs/webview/_xy-6521187_1]]
  Outbound Internet Access for ports 80/443 to the destination managing URL

Install Steps

To manually install agent software on an individual Windows or macOS endpoint:

1. Unzip the *.zip (Windows environments) or .dmg (macOS environments) file. The Windows agent installation package consists of these files: 

  Installation file:  xagtSetup_x.x.x_universal.msi
  Configuration file:  agent_config.json

2. Double-click the xagtSetup_x.x.x_universal.msi installation file to launch the setup wizard.

3. Accept all suggested settings and the license agreement, and continue through the wizard.

Other Notes

  1. Faculty laptops are eligible if they are university-owned. Personal devices are excluded.
  2. The HX management server is cloud-hosted, so there aren‚Äôt any on-campus dependencies. The only requirement is that the endpoint can establish an outbound connection to the HX server over the Internet.
  3. The installer can be passed to users for self-service installation.
  4. As of September 2021, UCOP covers 10,000 licenses of FireEye and according to Ryan, UCI has used a little less than half of the allocation so far.
  5. Contact Ryan Bakonis at rbakonis@uci.edu for inquiries relating to the console. Sub-delegation of console access is currently being worked on.

  7. Make sure to remove all other AV protection (such as Sophos) before enabling the AV portion of FireEye

ICS Host Sets

From Benny Angeles of OIT Security (November 7, 2024): 

Currently, there are several groups of hosts (also known as "host sets") in our Trellix console pertaining to ICS. They are:

1. ICS Domain (dynamic host set, currently 362 hosts)
2. ICS Host Prefixes (dynamic host set, currently 47 hosts)
3. ICS Machines (static host set, currently 12 hosts)

Dynamic host sets have the ability to automatically add hosts to themselves based on certain attributes of the host. 
For example, the "ICS Domain" dynamic host set grabs every host under the UCI-ICS domain that has the Trellix agent installed. 
So if you were to install the agent on a brand new machine under the UCI-ICS domain, once the agent communicates with the Trellix console, that host would automatically join the "ICS Domain" dynamic host set and obtain the custom policy applied to it.

Static host sets are sets of hosts that are manually created and that do not change, but custom policies can be applied to these as well.

The current configuration for the "ICS Domain" dynamic host set is:
-All hosts under the UCI-ICS domain

The current configuration for the "ICS Host Prefixes" dynamic host set is:
-All hosts that include the following strings in their hostname: Hostname~=/^BAO-|^COMM-|^CS-|^DEANS-|^GSAO-|^SAO-|^STATS-|^TEC-|^ER-|^INFO-/

The "ICS Machines" host set is a static host set so no dynamic configuration exists.

All 3 host sets are receiving the "ICS Workstations" custom policy.

If the configurations for these host sets need to be updated to include other hosts that are not currently included (i.e. add an additional domain, add specific AD OUs, etc) please let me know and I can work with you to update these.
