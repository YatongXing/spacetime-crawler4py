Project Rules

  1. Students may work in groups: Maximum size 3 students.
  2. Group members may change between projects.
  3. Project code to be submitted as a zip file to dropbox on due date. Project has multiple milestones. Each milestone to be submitted on time or "0" grade for that milestone.
  4. Project grade given based on: Playability 75%, User Guide 5% and Documentation 20%. Game demo during week 10 and finals period. All group members have to be present.
  5. Extra credit given for additional work.
  6. Copying strictly forbidden and if caught will result in a grade of "0" for all parties concerned.
  7. Have a USB drive of atleast 8GB to save your work in the lab.
