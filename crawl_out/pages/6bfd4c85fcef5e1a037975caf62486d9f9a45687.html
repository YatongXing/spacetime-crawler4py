Securing Open Services on Campus Computing

Stay Up to Date

  • Install OS updates frequently.
  • Reboot after every update to make sure the new software is installed.
  • Applying patches to CentOS 7
  • Ubuntu 16.04 apt-get update

Register your Servername.

Visit the following site to register your server with heldpesk:

  • https://netreg.ics.uci.edu

Once the hostname is registered ICS helpdesk will

  1. Assign you a permament statick IP address
  2. Register the hostname with the campus firewall

If you require ports open for services on an ICS host, please send an email to helpdesk@ics.uci.edu.

Configure Local Email

Make sure your local email service is configured correctly. Many of the above solutions will be much less effective, or completely ineffective if the local mailer is not setup correctly or if mail is ignored.

  • Install Postfix on CentOS 7
  • |Install Postfix on Ubuntu 16.04

Receive Results of Campus Security Scans

ICS and Campus regularly scan all hosts in the ICS domain. Send a note to helpdesk and request a copy of scans be sent to your mailbox.

Fail2Ban

Fail2ban scans log files (e.g. /var/log/apache/error_log) and bans IPs that show the malicious signs. Once a malicious client has been identified fail2ban uses the local firewall to block further connections from the client for a brief (configurable) period of time.

  • Fail2ban home
  • ICS Fail2ban information

Rootkithunter

Use the Rootkit Huner project package to notify you regarding suspicious system changes.

  • Rootkit Hunter Project
  • rkhunter @ ICS

Campus VPN

Use the campus VPN, or run your own, to limit the number of ports that are open to the internet at large and ensure the security of your data event when using an unsecured WIFI network.

  • Use the campus VPN to connect to your system from off campus, campus wifi network, or student housing.
  • Configure a self managed OpenVPN for improved ease of use.

Never Use Plaintext Protocols

  • Never use a plain-text protocol such as telnet, ftp, or rsh.
  • Always connect to a VPN first, make sure you connecsion traverse the VPN.

Securing MySQL

All off campus MySQL clients would first connect to the campus VPN before making a connection to a MySQL server. Opening MySQL services at the campus border is strongly discouraged because it exposes a attractive attack target and because data, including user credentials, are transmitted plaintext.

Secure MySQL

If using the campus VPN is not an option for some reason, consider using MySQL with openSSL. 

https://dev.mysql.com/doc/refman/5.7/en/secure-connections.html

A quick look at my own machine says that MySQL SSL is avaiable in the default mysql-server package for Ubuntu 16.04:

https://www.digitalocean.com/community/tutorials/how-to-configure-ssl-tls-for-mysql-on-ubuntu-16-04

Secure MySQL Access Control List

Since this system is exposed to the internet, please use very specific ACL's (e.g. 'non-root'@'128.195.14.151').

Secure with IPTables

Also prevent attacks by blocking all unknown connections to this port via iptables. Use rules similar to the these: 

iptables -A INPUT -p tcp -s <ip-address> --dport 3306 -j ACCEPT
iptables -A INPUT -p tcp --dport 3306 -j DROP
