ğŸ« Duo in ICS

ICS Computing Support uses Duo multi-factor authentication for sensitive systems and management tools.

Need Duo for your systems? Email helpdesk@ics.uci.edu if you are interested in using Duo on ICS systems or research equipment.

ğŸ“± Connecting Without the Phone App

Duo MFA settings on certain bastion or gateway hosts have been configured to accept the use of a DUO_PASSCODE to provide service to users with hardware tokens and poor phone connectivity.

Important: If you are required to use a passcode when connecting to a bastion host, please set up SSH keys to login to other ICS hosts without using Duo MFA.

ğŸ”‘ Duo Passcode Configuration

Users may force the auto-push behavior to their phone or tablet app by setting and exporting the following variable: 

export DUO_PASSCODE=push

Note: There is no longer a way to skip MFA entirely when using SSH keys on bastion hosts.

ğŸšª Login Process

Users enrolled in Duo will be presented with the following menu when logging in: 

Duo two-factor login for ____

Enter a passcode or select one of the following options:

 1. Duo Push to XXX-XXX-XXXX
 2. Duo Push to Android

Passcode or option (1-2):

Authentication Options:

  • Duo app users: Choose option 1 to receive a push notification to your phone
  • Hardware token/passcode users: Enter your 6-digit passcode directly

SSH Keys Note: All users logging into emp.ics.uci.edu will receive this prompt regardless of whether they are using SSH keys. Currently, this change only applies to emp.ics.uci.edu, and multi-factor authentication prompts will not appear on other hosts for users using SSH keys.

âš™ï¸ SendEnv DUO_PASSCODE Configuration

macOS or Linux SSH Client

In your environment, set the DUO_PASSCODE variable (Linux /bin/bash shown): 

export DUO_PASSCODE=push

Add the following to ~/.ssh/config or /etc/ssh/ssh_config: 

SendEnv DUO_PASSCODE

Note: This configuration has already been added to all ICS CentOS7 hosts.

Direct SSH Command Line

Use the -o SendEnv=DUO_PASSCODE option: 

ssh -o SendEnv=DUO_PASSCODE username@emp.ics.uci.edu

PuTTY Configuration

Navigate to Connection â†’ Data:

  • Enter DUO_PASSCODE as the variable
  • Enter push as the value

ğŸ”§ Additional Duo Settings

The Duo management interface includes four main sections:

  • Enrollment: Device setup and management
  • Test Token: Verify your authentication works
  • WebAuth Opt-In: Enable Duo for UCI WebAuth login
  • Office 365 Opt-In: Enable Duo for Office 365 login

Note: WebAuth and Office 365 Duo prompts are currently optional but may become mandatory in the future.

Required vs. Optional Duo Authentication

Duo host certificates come in two types:

  • Required Two-Factor: Users MUST use MFA to login
  • Opportunistic Two-Factor: Users MAY use MFA if enrolled

Configuration Issue: If a server set to â€œAllow Accessâ€ doesn't behave correctly, you may see this error: 

Aborted Duo login for '<user>' from host: Access Denied. 
The username you have entered cannot authenticate with Duo Security. 
Please contact your system administrator.

This requires Computing Support to contact OIT for configuration changes.

Account Lockout

Symptoms: Error in /var/log/secure on the target server: 

Mar 26 08:59:05 hostname sshd[31254]: Aborted Duo login for 'username' 
from client: Your account has been locked out due to excessive 
authentication failures. Please contact your administrator.
